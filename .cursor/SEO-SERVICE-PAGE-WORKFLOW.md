# SEO Service Page Workflow

Описание процесса (обзор шагов до/после вызова **seo-service-page-builder**) лежит в скилле **seo-service-page-workflow**:

| Где | Путь |
|-----|------|
| **В проекте (для git / команды)** | `.cursor/skills/seo-service-page-workflow/SKILL.md` |
| **Личный Cursor (копия)** | `~/.cursor/skills/seo-service-page-workflow/SKILL.md` |

Держите оба файла синхронными, если правите workflow.

**Имя скилла:** `seo-service-page-workflow`  
**Когда подключать:** вопросы вроде «как работает скилл SEO-страниц», «какие шаги у генерации страницы iRepair», обзор процесса до/после **seo-service-page-builder**.

**Генерация страниц:** скилл **seo-service-page-builder**  
**Вёрстка и SEO-нормы:** `.cursor/rules/seo--for-service-html-block.mdc`

### Обязательный шаг по картинкам

Перед генерацией страницы всегда задаётся вопрос: **«Нужны ли картинки для этой страницы?»**  

- Если пользователь отвечает **да** — запросить ссылки на изображения и добавить блок фото.
- Если пользователь отвечает **пропустить** или не даёт ссылки — **не добавлять** блок с картинками в страницу.
- Если пользователь просит картинки, но ссылки не прислал — повторно запросить ссылки; без ссылок блок не генерировать.
