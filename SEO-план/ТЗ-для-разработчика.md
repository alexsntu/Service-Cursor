### 1.3 jQuery — перенести в конец `<body>`

jQuery (`jquery.min.js`) сейчас в `<head>` без `defer` — это главный блокировщик первой отрисовки.

**Вариант А (рекомендуется):** Перенести тег jQuery прямо перед `</body>`:
```html
  <!-- ... весь контент страницы ... -->
  <script src="/catalog/view/javascript/jquery.min.js"></script>
  <!-- другие defer-скрипты подтянутся после -->
</body>
```

**Вариант Б (если перенос нарушает работу):** Добавить `defer`:
```html
<script src="/catalog/view/javascript/jquery.min.js" defer></script>
```

> ⚠️ После переноса jQuery — тщательно проверить работу калькулятора цен, форм и слайдеров.

## ЗАДАЧА 3 — Дополнения в robots.txt
**Приоритет: СРЕДНИЙ**  
**Срок: 1 день**

### Контекст
В текущем `robots.txt` уже хорошо прописаны многие блокировки. Но два типа URL проскользнули и попали в индекс Яндекса:
1. UTM-ссылки из Яндекс Директа: `/?utm_ya_campaign=76391371686&...`
2. Технические URL OpenCart: `/index.php?route=product/product&path=59_63...`

### Что добавить

Найти в `robots.txt` блок `# Блокировка index.php спама` и добавить строки:

```
# Блокировка UTM-параметров Яндекс Директа
Disallow: /*?utm_ya_campaign=
Disallow: /*&utm_ya_campaign=
Disallow: /*?utm_candidate=
Disallow: /*?yabizcmpgn=

# Блокировка технических URL OpenCart (дубли каталога)
Disallow: /index.php?route=product/product&path=

# Блокировка параметров Яндекс Метрики
Disallow: /*?_ym_status-check=
```

Добавить эти же строки во все три секции: `User-agent: *`, `User-agent: Google`, `User-agent: Yandex`.

**Важно:** Также добавить в секцию `User-agent: Yandex` директиву Clean-Param (если ещё нет):
```
Clean-Param: utm_ya_campaign&utm_candidate&yabizcmpgn irepair.ru
```
