### ШАГ A1 — Исправить 29 блокирующих JS-скриптов 👨‍💻
**Срок: 2–3 дня**  
**Статус: [ ] не начато**

**Зачем:**  
PageSpeed Score = 43/100, LCP = 31,8 сек. Причина — 29 из 31 скрипта загружаются синхронно и блокируют отрисовку страницы. Браузер ждёт каждый скрипт перед тем как показать хоть что-то на экране.

**Что делать:**

- [ ] Добавить `defer` ко всем скриптам в шаблоне OpenCart, кроме jQuery:
  ```html
  <!-- Было -->
  <script src="/catalog/view/javascript/swiper.min.js"></script>
  <!-- Надо -->
  <script src="/catalog/view/javascript/swiper.min.js" defer></script>
  ```
  Список скриптов для `defer`: swiper, TweenMax, scrollreveal, lottie, lightslider, lightgallery, lg-video, slick, spincrement, smoothscroll, smooth-scroll.polyfills, scroll-lock, scripts.js, alloka.js

- [ ] Добавить `async` к аналитике (не блокирует, не требует порядка):
  ```html
  <script src="https://www.googletagmanager.com/gtag/js?id=G-LSN9NP0Y82" async></script>
  <script src="/catalog/view/javascript/call.tracker.js" async></script>
  <script src="https://analytics.alloka.ru/script/397aa04f180fcb43" async></script>
  ```

- [ ] Удалить дублирующиеся скрипты (явные дубли в коде):
  - Оставить: `script.min.js` — удалить: `script.js`, `script(1).js`, `script.min(1).js`
  - Оставить: `inputmask.min.js` — удалить: `inputmask.dependencyLib.min.js` и `imask.js`
  - Оставить: `swiper.min.js` — удалить: `slick.min.js` (два слайдера одновременно не нужны)

- [ ] Добавить preload для критичных ресурсов в `<head>` перед всеми скриптами:
  ```html
  <link rel="preload" href="/catalog/view/javascript/app.css?v=1.39" as="style">
  <link rel="preload" href="/catalog/view/javascript/jquery.min.js" as="script">
  ```

- [ ] Включить Gzip-сжатие в `.htaccess` (если не включено):
  ```apache
  <IfModule mod_deflate.c>
    AddOutputFilterByType DEFLATE text/html text/css application/javascript application/json
  </IfModule>
  ```

- [ ] Включить браузерное кэширование в `.htaccess`:
  ```apache
  <IfModule mod_expires.c>
    ExpiresActive On
    ExpiresByType image/webp "access plus 1 year"
    ExpiresByType image/jpeg "access plus 1 year"
    ExpiresByType text/css "access plus 1 month"
    ExpiresByType application/javascript "access plus 1 month"
  </IfModule>
  ```

- [ ] Уточнить у хостинга: включён ли PHP OPcache (снизит TTFB с 0,9 до ~0,4 сек)

**Ожидаемый результат:** PageSpeed Score 43 → 70–80+, LCP 31,8 сек → 3–5 сек

### ШАГ A2 — Исправить страницу /error/ 👨‍💻
**Срок: 1 день**  
**Статус: [ ] не начато**

**Зачем:**  
Страница `/error/` занимает позицию 8,8 в Google, получает 11 603 показа и 126 кликов в месяц. Пользователи кликают и видят страницу с ошибкой — уходят, это портит поведенческие факторы.

**Что делать:**

- [ ] Добавить в шаблон страницы ошибки:
  ```html
  <meta name="robots" content="noindex, nofollow">
  ```
- [ ] Настроить редирект 301 в `.htaccess`:
  ```apache
  Redirect 301 /error/ https://irepair.ru/catalog/
  ```
- [ ] В GSC: раздел «Индексирование» → «Удаления» → «Новый запрос» → ввести `https://irepair.ru/error/`

**Ожидаемый результат:** Возврат 126 кликов в месяц на полезные страницы

### ШАГ A3 — Закрыть технические URL от индексации Яндекса 👨‍💻
**Срок: 1–2 дня**  
**Статус: [ ] не начато**

**Зачем:**  
В Яндексе проиндексированы 45 технических URL: рекламные UTM-ссылки и внутренние адреса OpenCart (`index.php?route=product/product&...`). Это дубли основных страниц, съедающие краулинговый бюджет.

**Что делать:**

- [ ] Добавить в `robots.txt` (файл в корне сайта):
  ```
  User-agent: *
  Disallow: /*?utm_
  Disallow: /*?_ym_
  Disallow: /index.php?route=product/product
  Disallow: /*?p=
  ```
- [ ] Проверить что `robots.txt` доступен: `https://irepair.ru/robots.txt`
- [ ] На 39 страниц `index.php?route=product/product&...` добавить canonical-тег, указывающий на соответствующую ЧПУ-страницу каталога
- [ ] В Яндекс Вебмастере → «Индексирование» → «Удаление страниц» → вручную удалить 3 UTM-URL

**Ожидаемый результат:** Освобождение краулингового бюджета, устранение дублей

**Срок: 3–7 дней**  
**Статус: [ ] не начато**

**Зачем:**  
33,6% сервисных страниц не имеют гео в Title. Запросы с «Москва» дают только 6,5% кликов из Яндекса при норме 30–50% для локального бизнеса.

**Что делать:**

  ```sql
  SELECT pd.product_id, pd.name, pd.meta_title 
  FROM oc_product_description pd
  WHERE pd.meta_title NOT LIKE '%осква%'
  AND pd.language_id = 1
  LIMIT 1000;
  ```

  1. Страницы категорий верхнего уровня (iPhone, MacBook, iPad, Watch, iMac)
